# IIS

Исследовательский проект по интегральному индексу состояния (ИИС) на основе физиологических данных `EEG`, `ECG` и `HRV`.

Текущая GitHub-версия репозитория собрана вокруг основной исследовательской линии:

- `V4` - первая честная негормональная база.
- `V6` - главное практическое strict-ядро и основной объект валидации.
- `V7` - инновационная экспериментальная ветка с `Sumigron`, пока не универсальная.

## Что находится в репозитории

- код расчета ИИС по нескольким версиям модели;
- загрузчики и извлечение признаков для открытых датасетов;
- сценарии запуска полного тестового пакета;
- GitHub-ориентированная markdown-документация;
- исходные `tex`-документы с полной математикой и расширенными материалами.

## Быстрый запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Пример запуска основного анализа:

```bash
python run_analysis.py --dataset ds002724 --mode strict --focus-version IISVersion6
```

Пример полного прогона:

```bash
python run_full_test_pack.py
```

Сборка итогового HTML/PDF-отчета:

```bash
python build_iis_full_report.py
```

## Базовая структура проекта

- `config/` - настройки проекта.
- `features/` - извлечение EEG и HRV признаков.
- `loaders/` - загрузчики датасетов.
- `models/` - версии ИИС и аналитические модули.
- `reporting/` - общая логика формирования отчетов.
- `docs/` - исходные научные документы и дополнительная документация.

## Что намеренно не хранится в Git

В репозиторий не включаются:

- локальные датасеты из `data/`;
- тяжелые результаты и графики из `outputs/`;
- временные каталоги из `temp/`;
- сгенерированные `pdf/html/log/aux/out`-артефакты.

Логика такая: репозиторий хранит код, структуру и документацию, а тяжелые данные и пересчитываемые артефакты воспроизводятся локально.

## Как использовать этот репозиторий в конкурсной упаковке

Для конкурсного текста и презентации лучше оставлять краткую и самодостаточную версию проекта, а в репозитории держать:

- расширенные формулы;
- полную историю развития версий;
- дополнительные результаты тестирования;
- технические детали воспроизводимости;
- отдельные интерпретационные документы.

Репозиторий должен быть расширением конкурсной версии, а не заменой основной работы.
