# GitHub-документация проекта ИИС

Этот раздел собирает GitHub-ориентированную документацию в markdown-формате. Он нужен для того, чтобы на репозиторий можно было ссылаться из конкурсных материалов без перегруза основной работы.

## Основные документы

- [Линия V4 -> V6 -> V7](model-line-v4-v6-v7.md)  
  Краткое объяснение, почему именно эти версии являются центральными для текущей упаковки проекта.

- [Формулы V4, V6 и V7](formulas-v4-v6-v7.md)  
  Ключевые формулы и математическая логика без полного переноса всего архива.

- [Сводка по валидации](validation-summary.md)  
  Основные датасеты, режимы проверки и аккуратные выводы по сильным и слабым местам модели.

- [Карта репозитория](repository-map.md)  
  Что находится в кодовой базе и за что отвечает.

## Принцип использования

Для конкурсной версии лучше работает следующая схема:

- в тексте и презентации остаются только ключевые формулы и основные результаты;
- в GitHub выносится расширенная документация;
- полные технические детали не подменяют собой основную исследовательскую аргументацию.
